Overview

Technology Stack

Four distinct technology layers orchestrated into a single ticketing platform.

⚛️ React

Single-page application host managing global state via Context API, hash-change routing, and interactive 3D seat animations with frosted-glass UI panels.

💚 Vue.js

Micro-widget containers mounted inside React DOM refs for modular card checkout forms and password audit consoles. Lifecycle managed via useEffect hooks.

☕ Java

Security engine implementing Luhn's card validation, private-field OOP encapsulation, ArrayList memory tracking, BufferedWriter file logging, and synchronized thread locks.

🐍 Python

Password audit layer deriving concrete rules (Length, Complexity, Pattern checks) from an abstract base class. CLI console emulator with Matplotlib stats visualization.

System Component Architecture

Fig 2.1 — CineBook Multi-Language Component Architecture

Component Layers
  • Frontend Client: React SPA with hash routing and context store. Vue containers mounted dynamically for checkout and audit widgets.
  • Java Security Layer: OOP encapsulation with Luhn's checksum validation, ArrayList object tracking, BufferedWriter disk logging, and mutex seat locks.

Architecture

Booking Sequence Flow

Fig 2.2 — End-to-End Booking Validation & Security Flow

Transaction Phases
1

Movie Selection & Seat Lock

Thread-0 acquires a synchronized mutex, locking selected seat coordinates for 5 minutes to prevent double-bookings.

2

Payment & Card Validation

Vue checkout collects card details. Luhn's mod-10 checksum validates digits. Valid cards are tracked in an ArrayList and logged to disk.

3

Password Security Audit

Dual validation: Java Weak/Strong classification + Python abstract auditor running LengthCheck, ComplexityCheck, and PatternCheck rules.

4

OTP Verification & 3D Ticket

6-digit OTP authentication → booking saved to LocalStorage → 3D parallax ticket with SVG QR code and scan verification.

Java Security Engine

// Luhn Algorithm — Card Number Validation public boolean checkLuhn(String cardNo) { int nDigits = cardNo.length(); int nSum = 0; boolean isSecond = false; for (int i = nDigits - 1; i >= 0; i--) { int d = cardNo.charAt(i) - '0'; if (isSecond) d = d * 2; nSum += d / 10; nSum += d % 10; isSecond = !isSecond; } return (nSum % 10 == 0); }
Java OOP Concepts Applied

🔒 Encapsulation

Private fields with public getter/setter methods. Card attributes are never directly exposed to external callers.

📊 ArrayList Tracking

Validated Card objects stored in an ArrayList. Memory allocation and heap positions are logged for visualization.

📁 BufferedWriter I/O

Transaction records serialized to cards.dat using Java's file I/O stream, ensuring persistent disk-level logging.

🔐 Thread Synchronization

Synchronized blocks on Thread-0 acquire mutex locks on seat coordinates, preventing concurrent double-bookings.

Python Audit Layer

Abstraction & Inheritance

🏗️ Abstract Base Class

PasswordCheck defines an abstract validate() method. Concrete subclasses (LengthCheck, ComplexityCheck, PatternCheck) override it with specific rules.

🔍 Regex Pattern Scan

Regular expressions detect repeated sequences, missing character classes (uppercase, digits, specials), and dictionary words.

📈 Matplotlib Visualization

Audit scores compiled into a bar chart rendered as PNG, embedded into the app's HTML5 canvas for real-time dashboard display.

Engineering

Challenges & Solutions

⚡ Double-Booking Race

Challenge: Simultaneous booking requests caused seat overlap conflicts across user sessions.

Solution: Implemented Java synchronized blocks with Thread-0 mutex locks. Seats are held for 5 minutes, blocking concurrent access.

🔗 React-Vue Collision

Challenge: Independent reactive frameworks caused DOM lifecycle conflicts when mounted side-by-side.

Solution: Managed mount/unmount cycles via React useRef + useEffect. Vue apps cleaned up on component destroy to prevent memory leaks.

🛡️ Exception Propagation

Challenge: Deep validation errors in Java/Python layers failed to surface in the frontend UI.

Solution: Structured custom exceptions with stack trace logging to sidebar console, enabling real-time developer debugging.
